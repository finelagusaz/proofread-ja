# 表記統一ルール

## 送り仮名の統一

### 基本ルール

送り仮名は「送り仮名の付け方」（内閣告示）に準拠する。

### 揺れやすい語

| 本則 | 許容 | 備考 |
|---|---|---|
| 行う | 行なう | 本則推奨 |
| 表す | 表わす | 本則推奨 |
| 現れる | 現われる | 本則推奨 |
| 断る | 断わる | 本則推奨 |

### 名詞化の送り仮名

| 動詞 | 名詞形 | 備考 |
|---|---|---|
| 受け付ける | 受付 / 受け付け | 名詞は省略可（統一すること） |
| 取り扱う | 取扱 / 取り扱い | 同上 |
| 申し込む | 申込 / 申し込み | 同上 |

## 漢字/ひらがな選択

### 基本方針

- 小説: 柔らかい印象を優先 → ひらがな多め
- 技術文書: 正確さを優先 → 漢字多め

### 判断が分かれる語

| 語 | 小説推奨 | 技術文書推奨 | 備考 |
|---|---|---|---|
| 事/こと | こと | 事 | 名詞化は「こと」優先 |
| 時/とき | とき | 時 | |
| 物/もの | もの | 物 | 抽象概念は「もの」 |
| 等/など | など | 等 | |
| 所/ところ | ところ | 所 | 形式名詞は「ところ」 |
| 様/よう | よう | よう | 形式名詞は「よう」 |
| 為/ため | ため | ため | 両方ひらがな推奨 |

※補助動詞のひらがな表記ルールは `common-errors.md` の「ひらがな表記推奨」セクション参照。

## カタカナ長音符

### 確認項目

- 全体で統一されているか
- 業界標準に従っているか

### 主な語例

| 長音あり | 長音なし | 備考 |
|---|---|---|
| コンピューター | コンピュータ | IT業界は長音なし多い |
| サーバー | サーバ | 同上 |
| ユーザー | ユーザ | 同上 |
| ブラウザー | ブラウザ | |
| フォルダー | フォルダ | |
| パラメーター | パラメータ | |

### 長音符を省略しないのが一般的な語

| 語 | 理由 |
|---|---|
| エラー | 2音節で短いため |
| データ | 定着した表記 |
| エリア | 定着した表記 |

## 英数字の全角/半角

### 数字

| 種類 | 推奨 | 例 |
|---|---|---|
| 技術文書の数値 | 半角 | 100個、2024年 |
| 小説の数値 | 全角または漢数字 | １００個、百個 |

### アルファベット

| 種類 | 推奨 |
|---|---|
| 英単語・略語・固有名詞 | 半角 |
| 強調目的 | 全角も可（稀） |

### 記号

| 記号 | 日本語文中 | 英数字のみ |
|---|---|---|
| 括弧 | 全角（） | 半角() |
| コロン | 全角： | 半角: |
| クエスチョン | 全角？ | 半角? |
| エクスクラメーション | 全角！ | 半角! |

## 句読点

### 種類の選択

| 種類 | 用途 |
|---|---|
| 。、（テン・マル） | 一般的な日本語文書 |
| ．，（ピリオド・カンマ） | 学術論文、公文書 |

文書内で統一すること。混合は避ける。

## スペースの扱い

| 場面 | ルール |
|---|---|
| 日本語と英数字の間 | スペースなし or 半角（統一する） |
| 英単語間 | 半角スペース |
| 日本語文中 | スペースなし |

## 括弧の使い分け

| 括弧 | 用途 | 例 |
|---|---|---|
| 「」 | 引用、発言、強調 | 彼は「行く」と言った。 |
| 『』 | 書名、作品名、「」内の引用 | 『源氏物語』を読んだ。 |
| （） | 補足説明、読み仮名 | 東京（とうきょう） |
| 【】 | 見出し、ラベル、強調 | 【重要】お知らせ |

### 括弧と句読点の位置

| パターン | 例 | 備考 |
|---|---|---|
| 句点は括弧の外 | 〜です（補足）。 | 一般的 |
| 括弧内が独立文 | （これは補足です。） | 括弧内にも句点 |
| 発言の終わり | 「行きます」 | 括弧内に句点なし |

## ダッシュ・波ダッシュの使い分け

| 記号 | Unicode | 用途 |
|---|---|---|
| 〜（波ダッシュ） | U+301C | 範囲（10〜20） |
| ～（全角チルダ） | U+FF5E | 波ダッシュの代用（避ける） |
| ー（長音符） | U+30FC | カタカナの長音 |
| ―（ホリゾンタルバー） | U+2015 | 言い換え・挿入 |

- 波ダッシュ（〜）と全角チルダ（～）は別の文字。波ダッシュを推奨
- 長音符（ー）を範囲の表現に使わない（×10ー20）

## 固有名詞の表記

### よくある表記の揺れ

| 公式表記 | 誤りやすい表記 |
|---|---|
| iPhone | Iphone、i-phone |
| macOS | MacOS、MACOS |
| JavaScript | Javascript、JAVASCRIPT |
| GitHub | Github、GITHUB |
| Wi-Fi | WiFi、Wifi、WIFI |

基本方針: 公式表記に従い、文書内で統一する。

## 文体の一貫性

### 敬体/常体

| 文体 | 用途 |
|---|---|
| 敬体（です・ます） | ブログ、マニュアル、ビジネス文書 |
| 常体（だ・である） | 論文、小説、報告書 |

### 許容される混在

| パターン | 例 |
|---|---|
| 引用内 | 「これは問題だ」と彼は言いました。 |
| 箇条書き | ・処理が高速である / 本文は「です・ます」 |
| 見出し | 【注意事項】/ 本文は「です・ます」 |

## 一人称の統一

| 一人称 | 印象 | 用途 |
|---|---|---|
| 私（わたし） | フォーマル、中性 | ビジネス、論文 |
| 僕 | カジュアル、男性的 | ブログ、エッセイ |
| 俺 | くだけた、男性的 | 小説の一人称 |
| 筆者/著者 | 客観的 | 論文、書籍 |

文書内で一人称が混在していないか確認する。
